## 4. Виды тестирования

### 4.1 Верификация

*Вопрос:* «Программа построена правильно?» (соответствие ТЗ разработчика)

| Проверка | Ожидание |
|----------|----------|
| Уникальный ISBN | Повторное добавление отклоняется |
| Поиск | Без учёта регистра, подстрока |
| Выдача/возврат | Флаг `available` меняется корректно |
| Список | Сортировка по названию |
| Корень `--version` | Возвращает 1.0.0 |
| Корень `--demo` | Код выхода 0 |

**Запуск:** `python tests/verification/test_verification.py`

### 4.2 Валидация

*Вопрос:* «Программа решает задачу пользователя с допустимыми данными?»

| Проверка | Ожидание |
|----------|----------|
| Корректный ISBN/год/текст | Успешное добавление |
| Пустое название, неверный ISBN | Отказ с сообщением |
| Год вне диапазона | Отказ |
| Неизвестный флаг CLI | Ненулевой код выхода |

**Запуск:** `python tests/validation/test_validation.py`

### 4.3 Юзабилити

*Вопрос:* «Программой удобно пользоваться?»

| Проверка | Ожидание |
|----------|----------|
| Справка `help` | Все команды, текст на русском |
| Ошибки | Понятные сообщения, не traceback |
| Пустой каталог | Сообщение «Каталог пуст» |
| Неизвестная команда | Подсказка про `help` |

**Запуск:** `python tests/usability/test_usability.py`
